## 发现的问题与修复

### 问题 1: Goroutine 泄漏
**描述**: cleanup goroutine 在测试结束后没有停止  
**影响**: 测试超时，资源泄漏  
**修复**: 添加 `stopChan` 和 `Stop()` 方法，支持优雅停止  
**状态**: ✅ 已修复

### 问题 2: HTTP 状态码错误
**描述**: 使用业务错误码 (1006) 作为 HTTP 状态码  
**影响**: HTTP 响应不符合标准  
**修复**: 使用标准 HTTP 429 (Too Many Requests) 状态码  
**状态**: ✅ 已修复

```yaml
  window: 1m
```

### 使用建议

1. **全局限流**: 使用 `RateLimitByIP` 防止 DDoS
2. **用户限流**: 使用 `RateLimitByUser` 防止滥用
3. **路径限流**: 对敏感接口使用 `RateLimitByPath`
4. **Redis 部署**: 生产环境建议使用 Redis 实现分布式限流
5. **监控日志**: 关注降级和恢复日志
